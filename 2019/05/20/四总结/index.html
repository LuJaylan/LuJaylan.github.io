###Swift VS Objective-C

  1. swift 注重安全性 Objective-C 注重灵活性
  2. swift 有函数式编程、面向对象编程、和面向协议编程，而Objective-C几乎只有面向对象编程
  3. swift 更注重值类型的数据结构，而Objective-C注重指针和索引。

    值类型高效利用内存：值类型是在栈上操作，仅仅是单个指针的移动。而引用类型是在堆上操作，牵涉合并、位移、重连接等。所以值内心能更高效地利用内存，并且copy-on-write又将值传递和复制的开销降到最低
    值类型线程安全：let设置使得数据达到真正意义上的不变，从根本上解决了多线程内存访问和操作顺序。
    提升了api灵活度，如实现collection协议，遍历string

  4. swift 是静态类型语言，而Objective-C是动态类型语言

ARC只是编译器在帮我们自动添加retain和release

###静态库和动态库

1.什么是库
库是共享程序代码的方式，一般分为动态库和静态库

2.动态库和静态库有什么区别
静态库：链接时完整地拷贝到可执行文件中，被多次使用就有多分冗余拷贝
动态库：链接时不复制，程序运行时由系统动态加载到内存，供程序调用，系统只加载一次，多个程序共用，节省内存

3.动态库和静态库的形式
动态库：.dylib和系统的.framework
静态库：.a和程序员自己建立的.framwork

.a是一个纯二进制文件，.framework中除了二进制文件之外还有资源文件
.a文件不能直接使用，至少要有.h文件配合，.framework文件可以直接使用
.a + .h + sourceFile = .framework
建议使用.framework

4.静态库存在的必要性

1.方便共享代码，便于合理使用
2.实现iOS程序的模块化，可以把固定的业务模块化成静态库
3.和别人分享你的代码库，但不想让别人看到你的代码实现
4.开发第三方sdk的需要

5.制作静态库时的几点注意

1.理解：无论是.a静态库还是.framework静态库，需要的都是二进制+.h+其他资源文件的形式，不同的是，.a本身就是二进制文件，需要我们自己配上.h 和其他资源文件才能使用，而.framework本身已经包含了.h和其它资源文件，可以直接使用
2.图片资源的处理：两种静态库，一般都是把图片文件单独的放在一个.bundle文件中，一般.bundle的名字和.a和.framework的名字相同。.bundle文件很好弄，新建一个文件夹，把它改名为.bundle就可以了，右键，显示包内容可以向其中添加图片资源
3.把category打成静态库，在用这个静态库的工程中，调用category中的方法时会找不到该方法的运行时错误（selector not recognized），解决办法是：在使用静态库的工程中配置other linker flags的值为-Objc
4.若一个静态库很复杂，需要暴露的.h比较多的话，可以在静态库内部创建一个.h文件，然后把所有需要暴露出来的.h文件都集中放在这个.h文件中，而那些原本需要暴露的.h都不需要再暴露了，只需要把.h暴露出来就可以了。

###CocoaPods
CocoaPods原理
将所有的依赖库放到另一个名为Pods项目中，然后让主项目依赖Pods项目，这样，源码管理工作都从主项目移到Pods项目中。技术细节有：

1.Pods项目最终会编译成一个名为libPods.a的文件，主项目只需要依赖这个.a文件即可。
2.对于资源文件，CocoaPods提供了一个名为Pods-resource.sh的bash脚本，该脚本在每次项目编译的时候都会执行，将第三方库的各种资源文件复制到目标目录中。
3.CocoaPods通过一个名为Pods.xcconfig的文件来编译时设置所有的依赖和参数。

pod 声明指定依赖的方法（指定一个工程的依赖）
podspec 提供一个简单的API用于创建podspecs
target 允许你在工程中限定依赖项只在你指定的target中生效

platform 指定了静态库应用被编译在哪个平台
user_frameworks 可以用framework的pod替代静态库的，可以通过自定义的子target来继承
source 指定specs的位置

category复写原类方法的原理

将category中的方法，属性，协议数据放在categ_t结构体中，然后将结构体内的方法列表拷贝到类对象的方法列表中。category可以添加属性，但并不会自动生成成员变量及set/get方法。因为category_t结构体中并不存在成员变量。通过之前对对象的分析我们知道成员变量是存放在实例对象中的，并且编译的那一刻就以及决定好了，而分类是在运行时才去加载的。那么我们就无法在程序运行时将分类的成员变量中添加到实例对象的结构体中。因此分类中不可以添加成员变量。

HTTP长连接了解吗

HTTP没有长短链接，HTTP是基于请求响应模式的，只要服务器给了响应，本次HTTP链接就结束了。
HTTP长连接和短连接本质是说的tcp链接。tcp链接是一个双向的通道，它是可以报酬已一段时间不关闭的，因此tcp链接才有真正的长连接和短连接之说。
把客户端的服务器的connection设置成keep-alive就会变成长连接（HTTP1.1默认是长连接）
长连接节省了tcp链接建立和断开的消耗

长连接并不是永久连接的。如果一段时间内（具体的时间长短，是可以在header当中进行设置的，也就是所谓的超时时间），这个连接没有HTTP请求发出的话，那么这个长连接就会被断掉。
